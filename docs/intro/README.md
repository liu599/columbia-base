# Columbia Base 项目介绍

## 📁 文件说明

本目录包含用于向同事介绍项目的两个核心文件：

1. **`api_collection.json`** - Postman API 集合文件
2. **`database_schema.sql`** - 数据库架构 SQL 文件

---

## 🚀 快速开始

### 导入 Postman Collection

1. 打开 Postman 应用
2. 点击左上角 "Import" 按钮
3. 选择 `api_collection.json` 文件
4. 导入完成后，您将看到所有 API 接口已按功能模块组织好

### 配置环境变量

在 Postman 中设置以下环境变量：

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `baseUrl` | API 基础地址 | `http://localhost:8080` |
| `token` | 普通 JWT 令牌 | 从登录接口获取 |
| `adminToken` | 管理 JWT 令牌 | 从管理员登录接口获取 |

## 🗄️ 数据库架构

### 核心表结构

#### 用户相关
- **`t_user`** - 用户基本信息
- **`t_user_auth`** - 用户认证令牌
- **`t_wechat_auth`** - 微信认证状态

#### 产品相关
- **`t_product`** - 产品信息
- **`t_activation_code`** - 激活码

#### 课程体系
- **`t_course`** - 课程基础信息
- **`t_chapter`** - 章节信息
- **`t_lesson`** - 课时内容

#### 用户学习数据
- **`t_user_course`** - 用户课程关联
- **`t_user_lesson_progress`** - 课时学习进度
- **`t_user_assignment`** - 作业提交记录

#### 积分系统
- **`t_credit_balance`** - 积分余额
- **`t_credit_log`** - 积分变动流水

#### 文件管理
- **`t_file`** - 文件元数据

#### 审计日志
- **`t_audit_log`** - 管理员操作记录

## 🔑 权限说明

### 用户角色等级

| 等级 | 角色 | 权限说明 |
|------|------|----------|
| 1 | 游客 | 基础浏览功能 |
| 3 | 正式用户 | 完整学习功能 |
| 5 | 付费用户 | 付费内容访问 |
| 7 | 内部管理用户 | 部分管理功能 |
| 10 | 系统管理员 | 所有管理功能 |

### 权限要求

- **普通用户接口**：需要登录（JWT Token）
- **管理员接口**：需要登录且角色等级 > 5

## 📝 注意事项

1. **接口版本**：当前版本为 v1 (`/api/v1/*`)
2. **认证方式**：JWT Bearer Token
3. **请求格式**：application/json
4. **响应格式**：统一的 ResultVo 格式
5. **时间格式**：ISO 8601 (YYYY-MM-DDTHH:mm:ss)
6. **分页参数**：`current` (当前页), `pageSize` (每页数量)

---

## 💡 开发建议

1. **本地开发**：将 `baseUrl` 设置为 `http://localhost:8080`
2. **测试环境**：根据实际环境修改 `baseUrl`
3. **Token 获取**：先调用登录接口获取 Token，然后设置到环境变量
4. **测试用例**：推荐使用 Postman Runner 进行批量测试
